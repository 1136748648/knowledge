# FR-VECTOR: 向量搜索模块需求

## FR-VECTOR-001: 向量插入
- **描述**: 将问答记录向量化并存储到Milvus
- **参与者**: 系统内部
- **前置条件**: 用户发起问答
- **主成功场景**:
  1. 用户提问
  2. 系统生成向量嵌入
  3. 存储到Milvus，附带元数据
- **拟合标准**: 向量插入成功，元数据完整（user_id, dept_id, sensitivity）
- **来源**: PRD用户故事#8
- **KANO类别**: 基本型
- **优先级**: P0
- **状态**: ✅ 已完成

---

## FR-VECTOR-002: 向量语义搜索
- **描述**: 根据用户提问进行语义相似度搜索
- **参与者**: 所有用户
- **前置条件**: 用户已登录，有访问权限
- **主成功场景**:
  1. 用户提交问题
  2. 系统生成向量嵌入
  3. 在Milvus中搜索相似向量
  4. 返回匹配结果
- **拟合标准**: 搜索响应时间<3秒，Top5结果相关度>80%
- **来源**: PRD用户故事#8
- **KANO类别**: 基本型
- **优先级**: P0
- **状态**: ✅ 已完成

---

## FR-VECTOR-003: 向量权限过滤
- **描述**: 搜索时根据用户权限过滤结果
- **参与者**: 系统内部
- **前置条件**: 用户已登录
- **主成功场景**:
  1. 用户发起搜索
  2. 系统根据dept_id和sensitivity过滤
  3. 返回用户有权限访问的结果
- **拟合标准**: 用户只能看到自己部门且敏感度允许的记录
- **来源**: PRD用户故事#9
- **KANO类别**: 基本型
- **优先级**: P0
- **状态**: ✅ 已完成
