# FR-DB: 员工数据库模块需求

## FR-DB-001: 员工档案查询
- **描述**: 根据姓名/工号/部门查询员工信息
- **参与者**: HR经理、部门经理
- **前置条件**: 用户已登录，有查询权限
- **主成功场景**:
  1. 输入查询条件
  2. 系统返回匹配的员工列表
  3. 点击查看详情
- **拟合标准**: 查询响应时间<1秒，结果正确过滤
- **来源**: PRD用户故事#4
- **KANO类别**: 基本型
- **优先级**: P0
- **状态**: ✅ 已完成

## FR-DB-003: 敏感字段脱敏
- **描述**: 根据用户角色对敏感字段（薪资、电话）进行脱敏处理
- **参与者**: 系统内部
- **前置条件**: 查询员工信息
- **主成功场景**:
  1. 用户查询员工信息
  2. 系统判断用户角色权限
  3. 对敏感字段进行脱敏显示
- **拟合标准**: 普通员工看不到薪资，部门经理看到本部门薪资脱敏
- **来源**: PRD用户故事#4
- **KANO类别**: 基本型
- **优先级**: P0
- **状态**: ⚠️ 部分实现

## FR-DB-004: 行级安全策略
- **描述**: 实现PostgreSQL行级安全（RLS）控制
- **参与者**: 系统内部
- **前置条件**: 数据库已配置RLS
- **主成功场景**:
  1. 用户发起查询
  2. PostgreSQL自动应用RLS策略
  3. 返回用户有权限访问的行
- **拟合标准**: 用户只能访问权限范围内的数据行
- **来源**: PRD用户故事#4
- **KANO类别**: 期望型
- **优先级**: P2
- **状态**: ⚠️ 部分实现（应用层过滤）
