# 多Agent合作的动态知识库系统设计（完整版）

## 一、详细设计步骤

### 1. 需求分析与目标定义
- **业务场景**：企业内部存在三类异构知识：制度文件（Wiki）、个人会话问答（非结构化QA历史）、员工档案（考勤、绩效、打卡等结构化数据）。
- **核心目标**：构建多Agent协同知识库平台，实现异构知识统一存储、智能检索、自动分类、审核优化、可视化，并在此基础上新增**登录验证、细粒度数据权限分级、可自动归纳的知识导航**三大前端能力。
- **关键能力清单**：
  - 规定文件以Wiki形式组织与浏览；
  - 个人会话问答存入向量数据库，支持语义检索；
  - 员工档案存入轻量关系数据库，支持精确查询与统计；
  - 跨库联合查询（如根据Wiki考勤制度和实际打卡记录计算扣款）；
  - 提交资料时自动判断存储环境、审核并优化内容；
  - 根据提示词生成结构化资料的思维导图；
  - 用户登录与身份认证；
  - 基于角色的数据访问控制，Agent查询时严格限制权限；
  - 多层级知识导航（企业资料、工作制度、项目资料等），支持手动创建与Agent自动归纳。

### 2. 系统总体架构设计（增强）
采用 **分层协同架构**，核心层扩展如下：
- **接入层**：Web提交维护页面（含登录）、会话问答界面、知识导航页、提示词输入接口。
- **安全与权限层（新增）**：统一身份认证（OAuth2.0/LDAP），权限策略引擎，数据访问控制过滤器。所有Agent调用底层存储前必须经过权限校验。
- **调度层**：中央调度Agent（Orchestrator），集成权限查询调用。
- **Agent协作层**：新增**权限管控Agent**、**分类与导航Agent**，原有Agent职责细化。
- **存储层**：Wiki引擎、向量数据库、轻量数据库，均强制绑定租户/部门/密级标签。
- **模型层**：LLM提供NL理解、分类、SQL生成等能力。

### 3. 多Agent角色定义与协作机制（增强版）
| Agent名称 | 职责 | 新增/增强能力 |
|-----------|------|---------------|
| **路由Agent** | 接收用户请求，进行意图识别和权限预检，指派执行Agent | 调用权限Agent进行操作许可验证 |
| **权限管控Agent**（新增） | 维护角色-数据映射，校验每次查询/写入的越权风险 | 基于RBAC/ABAC的实时鉴权；对敏感字段动态脱敏 |
| Wiki Agent | 管理Wiki文档CRUD、全文检索、版本控制 | 结合权限标签过滤页面可见性 |
| 向量Agent | 管理会话问答嵌入与语义搜索 | 用户/部门级数据隔离，仅召回授权范围内的会话 |
| 轻库Agent | 员工档案的条件查询与聚合计算 | 强制注入WHERE条件限定数据范围（如仅本部门或下属） |
| **审核优化Agent**（增强） | 内容合规检查、格式优化、**自动存储分类** | 可直接将文档归入知识导航的相应类目 |
| **分类与导航Agent**（新增） | 维护知识导航树，自动归纳未分类文档，提供导航数据 | 聚类/LLM生成分类标签，定期更新导航结构 |
| 思维导图Agent | 检索多源知识，生成思维导图JSON/Mermaid | 结合导航结构优化导图层级 |
| 协调Agent | 复合任务分解、工作流执行、结果聚合 | 尊重权限边界，跨库结果合并时标注权限受限部分 |

**权限校验协作流程**：所有存储操作请求（读/写）均由调度Agent统一拦截，转发至权限Agent进行鉴权，携带用户token与目标数据范围。权限Agent返回可执行的最小权限集合或拒绝原因。

### 4. 知识库存储详细设计（权限增强）
- **Wiki**：每个页面增加访问控制列表（ACL），支持按部门、角色、密级设置可见性。Agent通过API获取页面时携带用户身份，Wiki引擎过滤后返回。
- **向量数据库**：每条会话记录绑定`user_id`和`dept_id`，检索时通过标量过滤限定`dept_id IN (用户可见部门列表)`。
- **轻量数据库**：行级安全策略（Row-Level Security，如PostgreSQL RLS），或Agent生成SQL时强制追加`WHERE emp.dept_id IN (...) AND sensitivity <= user.clearance`。

### 5. 登录验证与数据权限分级设计
- **登录验证**：前端页面集成企业统一认证（OAuth2.0/LDAP），首次访问跳转登录，获取JWT token。后续所有API调用携带token。
- **权限模型**：基于RBAC，定义角色（普通员工、部门经理、HR、管理员等），每个角色对应数据访问规则：
  - 个人会话仅本人及直属上级可见；
  - 档案数据按“组织架构树”授权：普通员工仅看自己，经理看本部门及下级，HR可看全公司但敏感字段（如薪酬）加密显示；
  - Wiki文档按页面ACL控制读/写/管理权限；
  - 知识导航中部分类目（如“薪酬制度”）仅特定角色可见。
- **权限动态注入**：权限Agent维护一份数据授权策略配置文件，可热更新，无需重启服务。

### 6. 智能调用与跨库查询流程（权限融合示例）
以“张三本月考勤信息和迟到扣款”为例：
1. 路由Agent接收请求，提取当前用户身份（如部门经理李四），调用权限Agent。
2. 权限Agent校验：李四的管理范围包含张三 → 允许查询张三的考勤和档案；查询“考勤制度”Wiki页面为公开文档 → 允许。
3. 协调Agent分解任务，轻库Agent查询张三打卡记录时自动附加`dept_id`过滤，Wiki Agent提供制度文本。
4. 计算扣款结果返回，并显示依据来源链接。若用户无权查看扣款计算基准则部分隐藏。

### 7. 知识库知识导航设计（新增重点）
**导航结构**：树形分类，预设一级类目，支持无限层级扩展。
**预设类目（可配置）**：
- 企业资料（愿景、组织架构、证照）
- 工作制度（考勤、薪酬、行政）
- 项目资料（立项、进度、结项）
- 客户资料（客户信息、合同）
- 销售资料（报价、方案、话术）
- 运维资料（巡检、故障处理）
- 研发资料（技术规范、API文档）
- 软件工具（内部工具使用手册）

**导航维护方式**：
- **人工维护**：管理员可在导航管理页新建/编辑/移动节点，指定某个文档或问答归属。
- **自动归纳**：当用户提交资料时，审核优化Agent调用“分类与导航Agent”，基于内容语义推荐1~3个适合的导航类目。若置信度高于阈值则自动归类，否则提示用户选择。随着数据积累，分类模型持续微调。
- **定期重分类**：分类与导航Agent每月对未归类或分类置信度下降的条目重新评估，生成优化建议清单供管理员确认。

**前端展示**：
- 左侧导航树，点击节点展示该分类下的Wiki列表、精选问答、相关员工（如果是档案类导航）等聚合视图。
- 支持关键词搜索并自动定位到导航节点。

### 8. 提交维护页面设计增强
在原有基础上增加：
- **分类建议区**：实时显示Agent推荐的知识导航类目，用户可手动调整。
- **权限设置**：若用户有管理权限，可设置提交内容的可见范围（全员/指定部门/仅自己），否则默认继承分类权限。
- **审核优化区**：展示敏感词过滤、格式修正前后对比，以及最终的存储位置（Wiki/向量库/轻库）确认。

### 9. 思维导图生成增强
- 生成思维导图时，思维导图Agent可结合知识导航树的结构，将相关文档内容映射到对应的导航节点下，使导图与企业知识体系一致。
- 若请求涉及受限内容，导图相应节点标注“🔒权限不足”，不展示具体内容。

### 10. 技术选型补充
| 层次 | 技术组件 | 说明 |
|------|----------|------|
| 前端框架 | Vue3 + Vite | 构建登录、导航、提交、问答、导图页面 |
| 认证 | Keycloak / OAuth2-proxy | 统一登录与JWT签发 |
| 权限引擎 | Casbin / OPA | 细粒度策略管控 |
| 分类模型 | 自训练BERT + LLM few-shot | 新类目快速适配 |
| 导航存储 | PostgreSQL（邻接表/物化路径） | 与Wiki共库，方便关联 |

### 11. 实现步骤（含新增模块）
**第一阶段：基础存储与权限骨架**
- 搭建Wiki、向量库、轻量库。
- 集成登录认证，实现JWT签发与校验。
- 设计并实现基本RBAC，为三个存储库添加行级/文档级权限过滤。
- 开发路由Agent雏形，实现简单查询路由。

**第二阶段：导航与自动分类**
- 设计知识导航树结构并开发管理API。
- 训练/配置分类Agent，实现新资料自动归类。
- 开发审核优化Agent的格式优化与敏感词过滤。
- 实现知识导航前端界面，对接Wiki、向量搜索结果。

**第三阶段：多Agent协作与跨库查询**
- 实现权限Agent的细粒度动态鉴权。
- 构建协调Agent，完成考勤扣款等跨库复合查询。
- 全部Agent接入权限校验，确保无法越级访问。

**第四阶段：思维导图与完善**
- 开发思维导图生成Agent，整合知识导航结构。
- 增强维护页面，可视化权限设置与分类推荐。
- 整体测试、安全审计、用户培训、上线部署。

## 二、项目设计总结

本项目设计并实现了一套**安全可控、分级授权、自动归类**的多Agent协同动态知识库系统。在将Wiki制度文件、向量化会话记录、关系型员工档案三类异构数据统一纳管的基础上，重点强化了**身份认证、数据权限分级和智能化知识导航**三大企业级特性，使系统从“可查询”进化为“安全、有序、可探索”的企业认知中枢。

**核心价值创新：**
1. **全链路权限防护**：独创的权限管控Agent，确保每一次Agent发起的数据库查询、文档检索都经过精细化鉴权，数据访问遵循“最小够用”原则。行级安全、文档ACL、向量过滤等多重机制杜绝越权，满足企业合规要求。
2. **动态生长的知识导航**：预设业务导向的导航树结构，结合审核优化Agent与分类Agent的自动归纳能力，新知识进入系统即被智能归类，彻底解决了传统知识库“存而乱”的顽疾。人工与自动协同的维护方式，让知识体系随业务演进持续自我优化。
3. **异构存储下的统一视图**：导航树作为知识空间的索引，聚合了Wiki页面、精选问答、档案摘要等多种内容形态，用户从“搜”转变为“浏览+检索”，大幅提升知识发现效率。
4. **多Agent协作的可信查询**：跨库联合推理（如根据制度计算个人扣款）在权限约束下执行，全程透明标注信息来源与受限部分，既给出决策支持，又守住安全边界。
5. **从提交到可视化的闭环**：资料提交即经历分类、审核、优化、权限设定，最终沉淀至对应存储并自动挂载到导航节点；思维导图生成利用导航结构输出层次清晰的知识图谱，辅助培训与复盘。

**设计亮点：**
- 采用**Agent级权限中介模式**，将安全逻辑从业务逻辑中解耦，易于审计和策略变更。
- 知识导航树充当**多模态知识的结构化索引**，为RAG召回和思维导图生成提供了先验层级，提升生成内容的条理性。
- 提供自动分类与人工确认的弹性机制，贴合企业不同阶段的知识治理需求。

通过本方案，企业不仅获得一个高效的知识管理平台，更构建了一套**安全合规、持续进化、角色感知**的智能知识服务生态，为数字化决策、员工赋能、制度落地提供了坚实底座。
